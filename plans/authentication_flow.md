# Поток аутентификации

## Диаграмма последовательности

```mermaid
graph TD
    A[Пользователь] -->|Открывает форму регистрации| B[Фронтенд]
    B -->|Отправляет данные регистрации| C[Бэкенд: /api/users/register]
    C -->|Валидирует данные| D[База данных]
    D -->|Создает пользователя| C
    C -->|Возвращает статус| B
    B -->|Показывает сообщение| A

    A -->|Открывает форму входа| B
    B -->|Отправляет учетные данные| C[Бэкенд: /api/users/login]
    C -->|Проверяет учетные данные| D
    D -->|Возвращает пользователя| C
    C -->|Генерирует JWT токен| C
    C -->|Возвращает токен| B
    B -->|Сохраняет токен| B
```

## Процесс регистрации
1. Пользователь заполняет форму регистрации с номером участка, именем пользователя, email и паролем
2. Фронтенд отправляет данные на `/api/users/register`
3. Бэкенд валидирует данные:
   - Номер участка уникален
   - Email уникален
   - Пароль хешируется
4. Создается запись в базе данных
5. Возвращается статус операции

## Процесс входа
1. Пользователь вводит имя пользователя и пароль
2. Фронтенд отправляет данные на `/api/users/login`
3. Бэкенд проверяет учетные данные
4. При успешной проверке генерируется JWT токен
5. Токен возвращается фронтенду для дальнейших запросов
