# CLI Commands

## Создание суперпользователя

Для создания суперпользователя (администратора), который может активировать обычных пользователей, используйте одну из следующих команд:

### Вариант 1: Через переменные окружения (рекомендуется для автоматизации)

Раскомментируйте и заполните следующие переменные в файле `backend/.env`:

```bash
SUPERUSER_USERNAME=admin
SUPERUSER_PASSWORD=admin123
SUPERUSER_EMAIL=admin@example.com
SUPERUSER_PLOT_NUMBER=ADMIN-001
```

Затем запустите:

```bash
./create-superuser.sh
```

Или:

```bash
cd backend
python cli.py create-superuser
```

Команда автоматически возьмёт данные из `.env` и создаст суперпользователя без запросов.

### Вариант 2: Интерактивный режим

Если переменные не заданы в `.env`, команда запросит данные интерактивно:

```bash
./create-superuser.sh
```

При запуске вам будет предложено ввести:
- **Username** - имя пользователя для входа в систему
- **Email** - email адрес администратора
- **Plot number** - номер участка (может быть специальным значением для админа)
- **Password** - пароль (будет запрошен дважды для подтверждения)

### Вариант 3: Через параметры командной строки

```bash
cd backend
python cli.py create-superuser \
  --username admin \
  --email admin@example.com \
  --plot-number ADMIN-001 \
  --password secretpassword
```

### Приоритет параметров

Команда использует следующий приоритет:
1. Параметры командной строки (--username, --email, и т.д.)
2. Переменные окружения из `.env` (SUPERUSER_USERNAME, SUPERUSER_EMAIL, и т.д.)
3. Интерактивные запросы (если значения не найдены выше)

## Описание параметров

- **Username** - имя пользователя для входа в систему
- **Email** - email адрес администратора
- **Plot number** - номер участка (может быть специальным значением для админа)
- **Password** - пароль (будет запрошен дважды для подтверждения при интерактивном режиме)

## Результат

После успешного создания суперпользователя вы увидите сообщение:

```
✅ Superuser 'admin' created successfully!
   Role: admin
   Active: True
   Plot number: ADMIN-001
```

Созданный суперпользователь:
- Имеет роль **ADMIN**
- Автоматически **активирован** (is_active=True)
- Может входить в систему сразу после создания
- Имеет права на активацию обычных пользователей

## Просмотр неактивированных пользователей

Для просмотра списка пользователей, ожидающих активации:

```bash
cd backend
python cli.py list-inactive-users
```

Вывод будет содержать таблицу с информацией о пользователях:
- Username (имя пользователя)
- Email (email адрес)
- Plot Number (номер участка)
- Created At (дата регистрации)
